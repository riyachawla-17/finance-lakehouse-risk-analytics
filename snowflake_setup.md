### Snowflake Setup: Finance Lakehouse Analytics
Purpose:
- Define Gold analytics tables
- Grant access to Databricks loader user
- Provide validation queries

### Database & Schema

```
CREATE DATABASE IF NOT EXISTS FINANCE_LAKEHOUSE;
CREATE SCHEMA IF NOT EXISTS FINANCE_LAKEHOUSE.GOLD;
```

### Gold Tables (Analytics Layer)
```
CREATE OR REPLACE TABLE FINANCE_LAKEHOUSE.GOLD.KPI_DAILY (
  TXN_DATE DATE,
  CURRENCY STRING,
  COUNTRY STRING,
  TXN_COUNT NUMBER,
  TOTAL_AMOUNT FLOAT,
  AVG_AMOUNT FLOAT
);

CREATE OR REPLACE TABLE FINANCE_LAKEHOUSE.GOLD.CUSTOMER_RISK_DAILY (
  TXN_DATE DATE,
  CUSTOMER_ID STRING,
  TXN_COUNT_1D NUMBER,
  SPEND_1D FLOAT,
  AVG_AMOUNT_1D FLOAT,
  MAX_AMOUNT_1D FLOAT,
  HIGH_ACTIVITY_FLAG BOOLEAN,
  HIGH_SPEND_FLAG BOOLEAN,
  HIGH_VALUE_FLAG BOOLEAN,
  RISK_SCORE FLOAT,
  RISK_BAND STRING
);

CREATE OR REPLACE TABLE FINANCE_LAKEHOUSE.GOLD.REJECTS_DAILY (
  TXN_DATE DATE,
  REJECT_REASON STRING,
  REJECT_COUNT NUMBER
);
```
### Access Control for Databricks Loader
```
GRANT USAGE ON WAREHOUSE COMPUTE_WH TO USER DBX_LOADER;
GRANT USAGE ON DATABASE FINANCE_LAKEHOUSE TO USER DBX_LOADER;
GRANT USAGE ON SCHEMA FINANCE_LAKEHOUSE.GOLD TO USER DBX_LOADER;

GRANT SELECT, INSERT, UPDATE, DELETE
ON ALL TABLES IN SCHEMA FINANCE_LAKEHOUSE.GOLD
TO USER DBX_LOADER;
```
### Idempotent Reload Support
These TRUNCATE commands are used before reloading Gold tables from Databricks
```
TRUNCATE TABLE FINANCE_LAKEHOUSE.GOLD.KPI_DAILY;
TRUNCATE TABLE FINANCE_LAKEHOUSE.GOLD.CUSTOMER_RISK_DAILY;
TRUNCATE TABLE FINANCE_LAKEHOUSE.GOLD.REJECTS_DAILY;
```
### Validation Queries
```
SHOW TABLES IN SCHEMA FINANCE_LAKEHOUSE.GOLD;

SELECT COUNT(*) FROM FINANCE_LAKEHOUSE.GOLD.KPI_DAILY;
SELECT COUNT(*) FROM FINANCE_LAKEHOUSE.GOLD.CUSTOMER_RISK_DAILY;
SELECT COUNT(*) FROM FINANCE_LAKEHOUSE.GOLD.REJECTS_DAILY;

SELECT MIN(TXN_DATE), MAX(TXN_DATE)
FROM FINANCE_LAKEHOUSE.GOLD.KPI_DAILY;
```
